Canvas & Panel Size

1280 × 800 px — Landscape
No sidebar — full width content

1280 × 800 pixels, single-column layout

Same canvas size as the Multi-Display page — 1280 × 800 pixels, landscape, fixed. However, this layout removes the left sidebar entirely. The main content area is one full-width column, making it simpler and faster to operate in rooms where display assignment is not needed.

The three zones are the same:

  • Header bar — 80px tall, identical to all other pages.
  • Main content area — full width (~1280px), ~632px tall. Contains source grid and audio controls only.
  • Footer bar — 88px tall, identical to all other pages.

Use this layout for rooms with a single display or where the presenter only needs to pick a source and go — no routing required.

A deliberate, restrained palette

Every colour in this system has a specific purpose and should not be used arbitrarily. Here is what each one means:

  • Primary Blue (#2D49E8) — the header bar background, active interactive highlights, and slider controls. This is the brand anchor colour.
  • Blue Light (#EEF1FD) — used as the hover and focus background behind clickable items, giving a subtle blue wash without heavy contrast.
  • Active Green (#2D7D32) — fills a source button when that input is currently selected and routing to displays. Only ever used for "currently active" state.
  • Power Red (#B01A1A) — exclusively used for the shutdown/power button and the confirmation modal. Never use this colour for anything else.
  • Footer Dark (#34333B) — the charcoal background of the bottom navigation bar. Slightly warmer than pure black.
  • Panel BG (#F0F2FA) — the slightly blue-tinted grey used behind grouped content in left-side panels.
  • Button BG (#DDE1E7) — the default resting state of any tappable button. Neutral, low-contrast grey.
  • Border (#D1D8E0) — used for all card and button outlines. Keep borders consistent at 1.5px.
  • Text Dark (#111827) — primary label text on white or light backgrounds. Near-black for strong readability on touchscreens.

Segoe UI — four clear levels of hierarchy

Use Segoe UI across all panels. If unavailable, fall back to the device's system UI sans-serif font. Never mix typefaces. There are four type roles:

  • Section Heading — 16px / weight 700 / ALL CAPS / letter-spacing 0.08em / colour #111827. Used for top-level area labels like "SELECT SOURCE" or "AUDIO".
  • Button Label — 19px / weight 700 / sentence case / colour #111827 (white when active on green). Used for source names, scene names, and any primary tappable label.
  • Group Subtitle — 12–13px / weight 700 / ALL CAPS / letter-spacing 0.1em / colour #5a6080 (muted). Used inside cards to label sub-groupings like "Projector" or "Audience View".
  • Body / Status — 15px / weight 400 / colour #2d3748. Used for descriptive text, display names in lists, and status strings like "On" or "Off".

On a touchpanel, readability at arm's length is critical. Avoid any text below 12px in the UI itself.

Full width × 80px — Primary Blue bar

The header spans the full 1280px width and is exactly 80px tall. Background colour is #2D49E8. All text inside is white. Horizontal padding is 36px on each side.

The header is divided into three regions:

  • Left — Institution branding: A 48px white circle containing the initials "LU" in blue (14px / weight 800). Next to it, the institution name in 17px / weight 700 / uppercase, with the Māori name below at 11px / weight 400 / 80% opacity.
  • Centre — Room identifier: The room name or number at 22px / weight 600. This should be customised per room. In Crestron, bind this to a text join or hardcode the room name.
  • Right — Live clock: Current time displayed at 22px / weight 400, right-aligned. Update every minute via a digital join or serial join driven by the processor's real-time clock.

The header is identical across all pages in the system. Do not alter it per page.

3-column grid, filling the full content width

Because there is no left sidebar, the source buttons can span the full width of the panel. They are arranged in a 3-column grid with 12px gaps. Padding around the grid is wider than in the multi-display layout — 48px horizontal — since there is more room to breathe.

Source buttons fill the available height equally, dividing the space between the section heading and the audio divider. This makes them large and easy to tap on a touchscreen.

Button states are the same as the multi-display layout:

  • Default: #DDE1E7 background, #D1D8E0 border, dark icon and text
  • Active: #2D7D32 green, white icon and text
  • Blank all displays: Full-width row, dashed border. Turns #2D49E8 blue when selected.

Only one source can be active at a time. Tapping a new source clears the previous selection automatically.

7

Footer Navigation Bar

Full width × 88px — Charcoal navigation bar

The footer sits at the very bottom of the panel and is exactly 88px tall. Background is #34333B with a 2px top border of #28272E. It is the same on every page — only the active nav icon changes. It is divided into four zones from left to right:

  • Shutdown block (left): A full-height crimson block, approximately 100px wide, with a power icon and "SHUTDOWN" label stacked vertically. Tapping this triggers the Shutdown modal. Background is #B01A1A, darkening on hover. Deliberately separated from navigation.
  • Default block: A full-height #C89A00 yellow block, approximately 90px wide, with a reset/return arrow icon and "DEFAULT" label stacked vertically. Resets the room to default state. Icon and label are white (near-black) for legibility on the yellow background.
  • Navigation buttons (centre): 5 page buttons, each approximately 72px wide, stacked icon + label. Icons are white at 55% opacity when inactive; labels are 10px / weight 600 in the same muted white. The active page button gets a solid #E8E8E8 background with icon and label in dark colour (#111827). Pages: Display (monitor), Lighting (bulb), Camera (video), Audio (mic), Settings (gear).
  • Volume controls (right): Right-aligned group containing a mic-mute toggle with "Mic Mute" label and three volume icon buttons. When mic is muted, the icon and label turn #FF6B6B red.
The nav icon group is separated from volume controls by a 1px vertical divider at rgba(255,255,255,0.12). The same subtle divider separates the power block from the nav icons on the right side of the power block.

Full-canvas overlay — triggered by the power button only

When the operator taps the red power block in the footer, a modal overlay covers the entire 1280×800 canvas. This is an intentional full-screen interruption to prevent accidental shutdown. The overlay background is a dark semi-transparent layer (rgba(17,24,39,0.82)) with a subtle blur effect.

The white modal content is centred on screen and contains, from top to bottom:

  • A 96px circle with a light red tint background (#FDE8E8) and the power icon inside in red (#B01A1A). This reinforces the gravity of the action visually.
  • A title at 32px / weight 800 — "Shutdown System".
  • Body text at 20px explaining what will happen: room will shut down and all sessions end.
  • Two action buttons side by side, each with a maximum width of 240px and 16px border radius. "Yes, Shut Down" is filled red. "Cancel" is grey with a border. Both dismiss the modal — only "Yes" should trigger the actual shutdown sequence.

Do not allow any other tap target to dismiss this modal. The operator must explicitly choose one of the two buttons.

Pill-style slider — Master Volume only

Below the source buttons, a horizontal divider separates the audio control area. The audio section contains a single Master Volume control that expands to fill the full available width. The Mic Level row has been removed — microphone gain is managed at the system level, not from this panel.

The Master Volume row is laid out horizontally and contains:

  • Icon (left): A 22px speaker SVG icon drawn in the muted text colour (#5a6080) so it reads as supplementary, not primary.
  • Label: "Master Vol" at 15px / weight 700 / min-width 72px.
  • Pill control: A full-height rounded container (#F0F2FA background, #D1D8E0 border, 12px radius) that flexes to fill all remaining horizontal space. Contains three zones: a minus step button (52px wide, adjusts by −5%), a slider track with a blue circle thumb, and a plus step button (52px wide). No numeric value label is shown — the slider thumb position is the only level indicator.

The slider thumb is #2D49E8 blue with a 3px white border and a soft drop shadow. Tapping the step buttons is the primary interaction on a touchpanel; the slider itself is secondary for fine-tuning.
