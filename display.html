Inline SVGs based on Feather Icons

This UI uses vector iconography exclusively. The standard library used is Feather Icons (or its modern fork, Lucide). To ensure consistency, set viewboxes to 0 0 24 24, fill to none, and stroke-width to 2.

Canvas & Panel Size

1280 × 800 px — Landscape

1280 × 800 pixels, fixed landscape canvas

Set your Crestron project resolution to exactly 1280 × 800 pixels. This is a standard widescreen 16:10 ratio used on 10" touchpanels such as the TSW-1060 or TSW-1070. The layout is fully fixed — no scrolling occurs inside the panel itself.

The panel is divided into three horizontal zones stacked top to bottom:

  • Header bar — 60px tall. Blue background. Always visible, never changes.
  • Main content area — fills the remaining height (~632px). This area is split into two vertical columns: a 340px left sidebar and the remaining right area.
  • Footer bar — 88px tall. Dark charcoal background. Contains navigation and volume controls.

The diagram to the left shows this three-zone structure in miniature — blue header, light left sidebar, white right content, dark footer.

A deliberate, restrained palette

Every colour in this system has a specific purpose and should not be used arbitrarily. Here is what each one means:

  • Primary Blue (#2D49E8) — the header bar background, active interactive highlights, and slider controls. This is the brand anchor colour.
  • Blue Light (#EEF1FD) — used as the hover and focus background behind clickable items, giving a subtle blue wash without heavy contrast.
  • Active Green (#2D7D32) — fills a source button when that input is currently selected and routing to displays. Only ever used for "currently active" state.
  • Power Red (#B01A1A) — exclusively used for the shutdown/power button and the confirmation modal. Never use this colour for anything else.
  • Default Yellow (#C89A00) — exclusively used for the Default reset button in the footer. Icon and label use white or near-black for contrast. Never use this colour for general UI elements.
  • Footer Dark (#34333B) — the charcoal background of the bottom navigation bar. Slightly warmer than pure black.
  • Panel BG (#F0F2FA) — the slightly blue-tinted grey used behind grouped content in left-side panels.
  • Button BG (#DDE1E7) — the default resting state of any tappable button. Neutral, low-contrast grey.
  • Border (#D1D8E0) — used for all card and button outlines. Keep borders consistent at 1.5px.
  • Text Dark (#111827) — primary label text on white or light backgrounds. Near-black for strong readability on touchscreens.

Segoe UI — four clear levels of hierarchy

Use Segoe UI across all panels. If unavailable, fall back to the device's system UI sans-serif font. Never mix typefaces. There are four type roles:

  • Section Heading — 16px / weight 700 / ALL CAPS / letter-spacing 0.08em / colour #111827. Used for top-level area labels like "SELECT SOURCE" or "AUDIO".
  • Button Label — 16–19px / weight 700 / sentence case / colour #111827. Used for source names, display targets, scene names, and any primary tappable label.
  • Group Subtitle — 12–13px / weight 700 / ALL CAPS / letter-spacing 0.1em / colour #5a6080 (muted). Used inside cards to label sub-groupings like "Projector" or "Audience View".
  • Body / Status — 15px / weight 400 / colour #2d3748. Used for descriptive text and status strings.

On a touchpanel, readability at arm's length is critical. Avoid any text below 12px in the UI itself.

Full width × 60px — Primary Blue bar

The header spans the full 1280px width and is exactly 60px tall. Background colour is #2D49E8. All text inside is white. Horizontal padding is 28px on each side.

The header is divided into three regions:

  • Left — Institution branding: A 36px white circle containing the initials "LU" in blue (11px / weight 800). Next to it, the institution name in 15px / weight 700 / uppercase, with the Māori name below at 10px / weight 400 / 80% opacity.
  • Centre — Room identifier: The room name or number at 18px / weight 600. This should be customised per room. In Crestron, bind this to a text join or hardcode the room name.
  • Right — Live clock: Current time displayed at 18px / weight 400, right-aligned. Update every minute via a digital join or serial join driven by the processor's real-time clock.

The header is identical across all pages in the system. Do not alter it per page.

Default State Logic

DEFAULT

System Baseline (Prep Mode) Reset

The yellow "Default" block in the footer resets the room to a known, baseline state without shutting it down completely. This functions more like a "Lecture Prep Mode" rather than an absolute zero state.

Triggering the Default block explicitly executes the following specific overrides:

  • Master Volume: Snaps strictly to 80%. (Note: It is highly recommended to place a maximum output ceiling on the DSP side to ensure 80% is not painfully loud depending on the local PC source volume).
  • Projector: Blanked / No Source (Off).
  • Instructor & Front Displays: The Podium, Front Left, and Front Right displays are forced to show the Room PC source.
  • Rear Displays: Rear Left and Rear Right states remain unchanged from whatever the user had them set to previously.

The UI visually reflects these states immediately upon pressing the button by highlighting the "Room PC" source button and modifying the individual display cards.

Default Global Routing & Manual Override

The left column is explicitly labelled "Tap display to change source" to communicate that manual overrides are an optional, secondary workflow. Displays are grouped into white cards (Projector, Podium, Audience View).

Default Global Routing: Tapping a source button without explicitly selecting a display first will automatically route that source to all displays simultaneously. This ensures standard, room-wide changes are instant.

Manual Overrides: The left panel display buttons use the universal left-panel button style (16px bold, no chevron, 60px min-height). A selected display awaiting input always shows:

  • A full #2D49E8 blue fill with white text, regardless of whether it currently has an active source or is blank. This provides a perfectly consistent active state target.
  • Tapping a source then routes it only to the targeted display, bypassing the global default.

Source highlighting for immediate feedback

When a display is tapped in the left column, the Source grid automatically updates to highlight the current source actively routed to that specific display. If the display is showing "Room PC", the Room PC source button will instantly highlight in #2D7D32 green. If the display is blanked, the "Blank all displays" button will highlight in #2D49E8 blue. This provides the operator with instant, clear visual feedback of the system's routing state.

The Apply All button is positioned centrally below the source grid with reduced spacing. It perfectly matches the neutral grey resting state (#DDE1E7) and black text of the standard source buttons, acting as a clear but understated tool for global routing. It bypasses heavy animations or color changes, relying on a simple flat press interaction.

9

Footer Navigation Bar

Full width × 88px — Charcoal navigation bar

The footer sits at the very bottom of the panel and is exactly 88px tall. Background is #34333B with a 2px top border of #28272E. It is the same on every page — only the active nav button changes. It is divided into four zones from left to right:

  • Shutdown block (left): A full-height crimson block, approximately 100px wide, with a power icon and "SHUTDOWN" label stacked vertically. Tapping this triggers the Shutdown modal. Background is #B01A1A, darkening on hover. Deliberately separated from navigation.
  • Default block: A full-height #C89A00 yellow block, approximately 90px wide, with a reset/return arrow icon and "DEFAULT" label stacked vertically. Resets the room to default state. Icon and label are white (near-black) for legibility on the yellow background.
  • Navigation buttons (centre): 5 page buttons, each approximately 72px wide, stacked icon + label. Icons are white at 55% opacity when inactive; labels are 10px / weight 600 in the same muted white. The active page button gets a solid #E8E8E8 background with icon and label in dark colour (#111827). Pages: Display (monitor), Lighting (bulb), Camera (video), Audio (mic), Settings (gear).
  • Volume controls (right): Right-aligned group containing a mic-mute toggle with "Mic Mute" label and three volume icon buttons. When mic is muted, the icon and label turn #FF6B6B red.
The nav icon group is separated from volume controls by a 1px vertical divider at rgba(255,255,255,0.12). The same subtle divider separates the power block from the nav icons on the right side of the power block.

Full-canvas overlay — triggered by the power button only

When the operator taps the red power block in the footer, a modal overlay covers the entire 1280×800 canvas. This is an intentional full-screen interruption to prevent accidental shutdown. The overlay background is a dark semi-transparent layer (rgba(17,24,39,0.82)) with a subtle blur effect.

The white modal content is centred on screen and contains, from top to bottom:

  • A 96px circle with a light red tint background (#FDE8E8) and the power icon inside in red (#B01A1A). This reinforces the gravity of the action visually.
  • A title at 32px / weight 800 — "Shutdown System".
  • Body text at 20px explaining what will happen: room will shut down and all sessions end.
  • Two action buttons side by side, each with a maximum width of 240px and 16px border radius. "Yes, Shut Down" is filled red. "Cancel" is grey with a border. Both dismiss the modal — only "Yes" should trigger the actual shutdown sequence.

Do not allow any other tap target to dismiss this modal. The operator must explicitly choose one of the two buttons.

Consistent Left Panel Width Across All Pages

340px
340px

340px — fixed and identical on every page that has a left panel

Any page that uses the two-column layout (left context panel + right content area) must set the left column to exactly 340px. This applies to: Display (Multi), Camera, Lighting, and Audio. Pages without a left panel (Display Simple, Settings) are exempt.

Keeping the width identical across pages ensures the operator's eye always lands in the same place when switching pages. The left panel is a consistent navigational landmark — if it shifts width between pages, the UI feels broken even if every page works correctly in isolation.

  • Grid definition: grid-template-columns: 340px 1fr — never 300px, never 320px, never auto.
  • Padding inside the panel may vary slightly per page (e.g. 24px 20px vs 22px 18px) to accommodate different content densities — but the outer 340px column width must never change.
  • Border: Always 1.5px solid var(--border) on the right edge of the panel. This is part of the panel's visual footprint.
  • Background: Always var(--bg-panel) — the same #F0F2FA blue-tinted off-white used system-wide for secondary surfaces.

Do not widen this panel to accommodate more content. If a page has more items than fit, use internal scroll within the panel or reorganise content — never break the 340px rule.

The 340px width was chosen because it is the widest left panel needed (Display routing has the most items). All other pages adopt this same value so the layout grid never shifts between pages during normal operation.

Pill-style slider — Master Volume only

Below the source buttons, a horizontal divider separates the audio control area. The audio section contains a single Master Volume control that expands to fill the full available width. The Mic Level row has been removed — microphone gain is managed at the system level, not from this panel.

The Master Volume row is laid out horizontally and contains:

  • Icon (left): A 22px speaker SVG icon drawn in the muted text colour (#5a6080) so it reads as supplementary, not primary.
  • Label: "Master Vol" at 15px / weight 700 / min-width 72px.
  • Pill control: A full-height rounded container (#F0F2FA background, #D1D8E0 border, 12px radius) that flexes to fill all remaining horizontal space. Contains three zones: a minus step button (52px wide, adjusts by −5%), a slider track with a blue circle thumb, and a plus step button (52px wide). No numeric value label is shown — the slider thumb position is the only level indicator.

The slider thumb is #2D49E8 blue with a 3px white border and a soft drop shadow. Tapping the step buttons is the primary interaction on a touchpanel; the slider itself is secondary for fine-tuning.
